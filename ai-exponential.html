一、填空题
第1题：数据集的三个重要特性：______、______和______。

正确答案：维度、稀疏性、分辨率

详细解析：

  • 维度（Dimensionality）：数据集的属性个数，即特征向量的维度，维度越高数据处理复杂度越高
  • 稀疏性（Sparsity）：数据集中零值或缺失值所占的比例，高稀疏性数据需要特殊的存储与处理算法
  • 分辨率（Resolution）：数据的精细程度，如采样频率、量化级别，分辨率越高数据越精确但数据量越大
第2题：数据采集的三大要点：______、______和______。

正确答案：全面性、多维性、高效性

详细解析：

  • 全面性：尽可能覆盖所有相关数据，避免样本偏差，保证数据能够反映事物全貌
  • 多维性：从多个维度采集数据，避免单一维度的片面性，支持多维度交叉分析
  • 高效性：在有限资源下，快速采集有价值的数据，避免冗余数据占用存储与算力
二、公式题
第3题：标准分数（Z-score）公式
z = (x - μ) / σ

参数说明：x = 原始数据值，μ = 总体均值，σ = 总体标准差

用途：将不同量纲的数据标准化，消除量纲影响，便于不同指标之间的比较与综合评价

第4题：离散系数（变异系数）公式
CV = (σ / μ) × 100%

参数说明：σ = 标准差，μ = 均值

用途：比较不同量纲数据的离散程度，消除量纲与均值水平的影响，CV越大说明数据离散程度越高

三、思考题
第5题：100个人年收入1000万，均值能反应这个人群的收入特征吗？为什么？

结论：不能。

详细解析：

  1. 极端值对均值影响极大：如果100人中有1个亿万富翁，其收入会严重拉高整体均值，使均值无法反映绝大多数人的真实收入水平
  2. 收入分布属于右偏分布：收入数据通常呈现少数高收入者拉高整体水平的特征，均值会向高收入端偏移
  3. 中位数更适合描述收入特征：中位数不受极端值影响，能更好反映收入的集中趋势与大多数人的真实水平
  4. 总结：对于存在极端值的偏态分布数据，中位数比均值更具代表性，均值仅适合正态分布数据的集中趋势描述

五、思考题（续）
第10题：数据的属性：标称、序数、区间、比率。判断手机型号、成绩等级、温度、电压分别属于什么属性。
  • 手机型号 → 标称属性：仅用于分类，类别之间无顺序与大小关系
  • 成绩等级 → 序数属性：类别之间有明确顺序（优秀>良好>及格），但差值无实际意义
  • 温度 → 区间属性：差值有实际意义（20℃比10℃高10℃），但无绝对零点（0℃不代表没有温度）
  • 电压 → 比率属性：有绝对零点（0V代表没有电压），比值有实际意义（10V是5V的2倍）
第11题：分析开关状态、光照强度、温湿度等传感器数据的属性类型（离散or连续）。
  • 开关状态 → 离散数据：仅能取0/1两个有限的离散值，无中间值
  • 光照强度 → 连续数据：可取0到最大值之间的任意实数值，取值连续无间隔
  • 温湿度 → 连续数据：温度、湿度均可取一定范围内的任意实数值，属于典型的连续型传感器数据
第12题：电子信息工程中"模拟信号采样得到的数字数据"属于定性还是定量数据？说明离散型与连续型数据在专业场景中如何区分，举例（传感器电压数据、通信协议帧计数数据）。

结论：属于定量数据。

离散型与连续型数据区分方法：

  • 传感器电压数据 → 连续型数据：电压可取任意实数值，取值连续，存在中间值，属于连续型定量数据
  • 通信协议帧计数数据 → 离散型数据：帧计数只能取整数值，不存在小数帧，取值是离散的，属于离散型定量数据

区分核心：连续型数据可取任意实数，离散型数据只能取有限或可数的数值（通常为整数）。

六、计算题
第13题：计算电压数据[2.1, 2.3, 2.2, 2.3, 2.5, 1.9, 2.3, 2.4]的均值、中位数、众数。说明异常值下中位数的稳健性。

【步骤1：计算均值】

数据求和 = 2.1 + 2.3 + 2.2 + 2.3 + 2.5 + 1.9 + 2.3 + 2.4 = 18.0

均值 = 18.0 ÷ 8 = 2.25V

【步骤2：计算中位数】

数据排序后：[1.9, 2.1, 2.2, 2.3, 2.3, 2.3, 2.4, 2.5]

第4个值 = 2.3，第5个值 = 2.3

中位数 = (2.3 + 2.3) ÷ 2 = 2.3V

【步骤3：计算众数】

2.3出现3次，出现次数最多

众数 = 2.3V

【中位数的稳健性说明】

中位数仅与数据的排序位置有关，不受极端异常值影响；而均值对异常值高度敏感，若数据中加入一个10V的异常值，均值会变为3.11V，严重偏离真实值，但中位数仍为2.3V，体现了中位数的强稳健性。

第14题：结合"数据采集误差"（传感器漂移、传输干扰异常值），讨论：均值、中位数、众数哪个更适合描述存在异常值的专业数据？为什么？

结论：中位数最适合描述存在异常值的工程数据。

核心论述：

  1. 均值的局限性：均值对异常值高度敏感，传感器漂移、传输干扰产生的极端异常值会严重拉高或拉低均值，使均值无法反映数据的真实集中趋势
  2. 众数的适用性低：众数仅反映出现频率最高的数值，工程传感器数据多为连续型数据，众数无实际工程意义，且可能不存在或存在多个众数
  3. 中位数的核心优势：中位数是数据排序后的中间值，仅与数据位置有关，不受极端异常值影响，稳健性最强，非常适合存在干扰的工程传感器数据

专业案例：某电流传感器采集的10组数据：[1.2,1.3,1.2,1.4,1.3,1.2,1.3,10.0,1.4,1.3]，均值为2.16A（受10.0A异常值严重干扰），中位数为1.3A（完全匹配真实工作电流），验证了中位数的稳健性。

第15题：100个嵌入式设备故障：硬件35、软件42、通信18、人为5。计算异众比率，说明数值含义及工程参考意义。

【计算过程】

众数类别为"软件故障"，频数=42，总频数=100

异众比率 = 1 - (众数频数 ÷ 总频数) = 1 - (42 ÷ 100) = 0.58

【数值含义】

58%的故障不属于众数类别"软件故障"，说明嵌入式设备的故障类型较为分散，不存在占绝对主导的故障类型。

【工程参考意义】

异众比率越高，说明数据越分散，众数的代表性越差。本案例中异众比率0.58，说明不能仅针对软件故障优化，硬件、通信故障也需要同步优化，需全面提升设备可靠性。

第16题：结合通信工程中的"信号抖动""噪声强度"，说明方差/标准差的意义。

核心定义：方差是数据与均值偏离程度的平方和的均值，标准差是方差的平方根，二者均反映数据的离散程度，标准差单位与原始数据一致，更便于工程解读。

通信工程场景解读：

  1. 信号抖动维度：信号抖动指信号传输过程中时域上的相位偏移与幅度波动，方差/标准差越大，说明信号抖动越剧烈，信号的时域稳定性越差，会导致通信误码率上升
  2. 噪声强度维度：通信信道中的热噪声、电磁干扰噪声会导致采样值偏离真实值，方差/标准差直接量化噪声强度——标准差越大，噪声越强，信噪比（SNR）越低，通信质量越差

工程应用：工业控制通信中，4-20mA电流信号的标准差需＜0.1mA，否则会导致执行器动作误差超标；射频通信中，接收信号强度（RSSI）标准差＞3dB时，说明链路噪声过大，需优化信道。

第17题：30个学生成绩：60,65,68,70,72,74,75,76,77,78,79,80,80,80,81,82,82,83,84,85,85,85,86,88,88,90,92,95,98,100，求异众比率和离散系数。

【步骤1：计算异众比率】

众数为80和85，均出现3次，取众数频数=3

异众比率 = 1 - (3 ÷ 30) = 0.9

含义：90%的成绩不属于众数类别，说明学生成绩分布非常分散，没有明显的集中分数段。

成绩标准差 ≈ 9.85分

第18题/第25题：校园人流量统计系统可选择哪些数据采集方式？各有什么优缺点？

校园人流量统计系统常用三种数据采集方式，优缺点如下：

1. 视频AI视觉采集

原理：通过摄像头+目标检测算法识别跟踪行人，实现计数与方向识别

优点：覆盖范围广、可区分进出方向、数据维度丰富（支持热力图、轨迹分析）、可联动安防功能

缺点：隐私风险高、受光照/遮挡影响大、部署与算力成本高、极端环境准确率下降

2. 红外对射采集

原理：通过红外对射管的光束通断触发计数

优点：成本极低、安装简单、无隐私风险、环境适应性强、响应速度快

缺点：无法识别方向、多人并排易漏检、统计误差大、功能单一、仅适合窄通道

3. WiFi探针采集

原理：被动采集智能设备的WiFi探测帧，基于MAC地址计数

优点：可复用校园WiFi网络、无感知采集、支持停留时长与轨迹分析、覆盖范围大

缺点：存在系统性偏差（仅统计开WiFi的设备）、无法区分人与设备、信号易受干扰、存在隐私合规风险

第19题/第26题：常见传感器在大数据采集场景中会遇到哪些典型问题？

传感器大数据采集的典型问题包括：

  1. 传感器漂移：长期工作后传感器零点与灵敏度发生偏移，导致数据系统性偏差
  2. 噪声干扰：电磁干扰、环境噪声导致数据出现随机波动，降低数据质量
  3. 数据不同步：多传感器采集的时序数据时间戳不统一，无法进行时序关联分析
  4. 数据丢包：无线传输过程中数据丢失，导致时序数据断裂
  5. 功耗限制：无线传感器电池容量有限，需在采样频率与续航之间平衡
  6. 电磁干扰：工业环境中强电磁干扰导致传感器输出异常值
  7. 环境适应性差：温湿度、气压等环境变化影响传感器测量精度
第20题/第27题：采集用户行为数据时有哪些隐私与伦理问题？如何平衡？

隐私与伦理问题：

  1. 未经用户知情同意采集个人行为数据，侵犯用户知情权
  2. 过度采集与业务无关的个人敏感数据，存在数据滥用风险
  3. 数据存储与传输过程中存在泄露风险，可能导致用户隐私泄露
  4. 用户行为数据被用于用户画像与精准推送，存在"大数据杀熟"等伦理问题
  5. 数据匿名化不彻底，可通过多维度数据关联还原用户身份

平衡方法：

  1. 用户知情同意：明确告知用户数据采集的目的、范围与使用方式，获得用户授权
  2. 数据脱敏：对姓名、手机号、身份证号等敏感信息进行脱敏处理
  3. 差分隐私：在数据中加入噪声，在保证数据分析有效性的同时保护用户隐私
  4. 联邦学习：数据不出本地，仅传输模型参数，实现数据可用不可见
  5. 最小必要原则：仅采集业务必需的数据，不过度采集用户信息
第21题/第28题：传感器原始时序数据包含异常点、缺失值、噪声，说出三种处理方法。
  1. 异常点处理：3σ原则

    计算数据的均值μ与标准差σ，将超出[μ-3σ, μ+3σ]范围的数据判定为异常值，进行剔除或修正，可过滤99.7%的随机异常点

  2. 噪声处理：中位数滤波

    对时序数据采用滑动窗口，用窗口内数据的中位数替换当前值，可有效去除脉冲噪声，同时保留数据的边缘特征，非常适合传感器去噪

  3. 缺失值处理：线性插值

    对于缺失的时序数据，利用缺失点前后的两个有效值进行线性插值，填补缺失值，保证时序数据的连续性，适合变化平缓的传感器数据

  4. 进阶方法：卡尔曼滤波

    结合系统状态方程与观测方程，可同时实现去噪、缺失值填补与状态预测，是工业时序数据处理的高级方法

第23题/第30题：数据采集、数据处理、数据可视化，哪一步对AI模型效果影响最大？

结论：数据处理对AI模型效果影响最大。

核心原因：

  1. 数据质量决定模型上限："垃圾进，垃圾出"，即使是最优秀的算法，使用低质量数据训练也无法得到好的模型效果
  2. 数据处理是承上启下的关键环节：数据采集得到的原始数据存在大量噪声、异常值、缺失值，数据处理是将原始数据转化为模型可用的高质量数据的核心步骤
  3. 特征工程属于数据处理范畴：AI模型效果70%以上取决于特征工程，而特征工程是数据处理的核心内容，好的特征可以大幅提升模型效果
  4. 数据采集仅决定数据的丰富度，数据可视化仅决定结果的展示效果，二者都不直接决定模型的最终效果
第24题：采样率太低或太高，分别会对数据造成什么影响？

采样率太低的影响：

  1. 信息丢失：无法捕捉信号的高频变化细节，导致关键特征丢失
  2. 频谱混叠：采样率低于奈奎斯特采样频率时，会发生频谱混叠，无法还原原始信号
  3. 分析精度下降：时序分析、故障诊断等应用的精度会大幅降低

采样率太高的影响：

  1. 数据量过大：产生大量冗余数据，占用大量存储与传输带宽
  2. 引入冗余噪声：过高的采样率会引入高频噪声，反而降低数据质量
  3. 处理算力浪费：数据量过大导致模型训练与数据处理的算力成本大幅上升
  4. 传感器功耗上升：高采样率会大幅增加无线传感器的功耗，缩短续航时间

一、填空题（补充考试重点）
填空题1：云计算的七大核心特点：______、______、______、______、______、______、______。

正确答案：超大规模、虚拟化、高可靠性、通用性、高可伸缩性、按需服务、极其廉价

详细解析：

  • 超大规模：云厂商拥有数十万甚至百万级服务器集群，提供远超单个企业的计算能力
  • 虚拟化：通过虚拟化技术实现资源池化，用户可按需获取虚拟资源，无需关心物理硬件
  • 高可靠性：多副本容错、节点自动切换、数据多地域备份，可靠性远高于本地机房
  • 通用性：不针对特定应用，同一云平台可支撑千行百业的不同业务场景
  • 高可伸缩性：可根据业务负载弹性扩缩容，秒级调整资源规模，应对流量高峰
  • 按需服务：用户按需申请资源，按实际使用量付费，像水电一样计量计费
  • 极其廉价：规模效应带来成本优势，用户无需承担高额的硬件采购与运维成本
填空题2：云计算按照服务模式分为三类：______、______、______。

填空题3：虚拟化的四大核心特点：______、______、______、______。

正确答案：分区、隔离、封装、相对硬件独立

概念详细解释：

  • 分区（Partitioning）：在一台物理服务器上可同时运行多个虚拟机，将物理硬件资源划分为多个逻辑资源，每个虚拟机获得独立的计算、内存、存储资源，实现资源的复用
  • 隔离（Isolation）：虚拟机之间完全隔离，一个虚拟机的崩溃、故障或安全问题不会影响其他虚拟机，故障域隔离，保障业务稳定性
  • 封装（Encapsulation）：整个虚拟机的运行环境（操作系统、应用、配置、数据）可封装为一个或多个文件，便于备份、迁移、克隆和快照恢复
  • 相对硬件独立（Hardware Independence）：虚拟机与底层物理硬件解耦，可在不同品牌、不同型号的物理服务器之间迁移，无需修改虚拟机内部配置
填空题4：虚拟化按照实现类型分为三类：______、______、______。

正确答案：寄居虚拟化、裸金属虚拟化、混合虚拟化

概念详细解释：

  • 寄居虚拟化（Type 2）：VMM作为应用程序安装并运行在宿主操作系统之上，依赖宿主OS提供硬件驱动和资源管理，典型产品：VMware Workstation、VirtualBox；特点：安装简单、兼容性好、适合个人用户，性能有损耗
  • 裸金属虚拟化（Type 1）：VMM直接安装运行在物理硬件之上，无需宿主操作系统，直接管理硬件资源，典型产品：VMware ESXi、KVM、Xen；特点：性能高、隔离性强、适合企业级数据中心
  • 混合虚拟化：结合寄居与裸金属的优势，核心部分直接运行在硬件上，部分功能借助宿主OS，兼顾性能与易用性，是部分现代虚拟化产品的优化方向
填空题5：虚拟化按照架构分为三类：______、______、______。

正确答案：全虚拟化、半虚拟化、硬件辅助虚拟化

概念详细解释：

  • 全虚拟化：通过二进制翻译技术模拟完整的硬件，Guest OS完全无需修改，不知道自己运行在虚拟机中；特点：兼容性最好，支持所有操作系统，性能有一定损耗
  • 半虚拟化：需要修改Guest OS内核，让操作系统主动配合VMM进行通信，避免二进制翻译开销；特点：性能极高，接近物理机，兼容性差，仅支持开源可修改内核的系统（如Linux）
  • 硬件辅助虚拟化：CPU层面提供虚拟化硬件支持（Intel VT-x、AMD-V），VMM可直接执行敏感指令，无需二进制翻译；特点：兼顾性能与兼容性，是当前主流的虚拟化架构
填空题6：集中式存储主要指的是______。

正确答案：磁盘阵列（RAID）

概念解释：集中式存储是将所有数据集中存储在独立的磁盘阵列存储设备中，所有服务器通过网络访问统一的存储池，核心设备就是磁盘阵列（RAID），通过多块磁盘组合实现高性能、高可靠的统一存储。

二、云计算基础（第1-3题）
第1题（思考讨论题）：你平时用过哪些"云服务"?它们有什么特点?它们为什么算"云计算"?

日常常用云服务举例：

  • 存储类：百度网盘、阿里云盘、腾讯微云
  • 办公类：腾讯文档、飞书、钉钉、在线Office

  • 生活类：网易云音乐、在线视频网站、云游戏

云服务共同特点：

  1. 网络访问：无需本地安装，通过网络即可随时随地访问使用
  2. 按需自助：用户可自行开通、扩容、取消服务，无需人工干预
  3. 资源共享：多用户共享云厂商的服务器集群资源，成本分摊
  4. 弹性伸缩：可根据用户需求动态调整资源，高峰期自动扩容
  5. 可度量服务：按实际使用量计费，用多少付多少

为什么属于云计算：完全符合云计算的五大核心特征：按需自助服务、广泛网络访问、资源池化、快速弹性伸缩、可度量的服务，因此属于典型的云计算应用。

第2题（思考讨论题）：云计算和自己买服务器搭机房,核心区别是什么?两者各有什么优势?

核心区别：CAPEX（资本支出）vs OPEX（运营支出）

自建机房是一次性大额资本支出（买硬件、建机房），云计算是持续小额运营支出（按使用量付费）；本质是"买"vs"租"的区别。

云计算优势：

  1. 成本优势：无需前期大额硬件投入，按需付费，避免资源闲置浪费
  2. 弹性优势：秒级扩缩容，轻松应对业务高峰，无需提前规划容量
  3. 运维优势：厂商负责硬件运维、机房管理、故障修复，无需专业运维团队
  4. 上线速度：分钟级开通资源，业务快速上线，无需等待硬件采购部署
  5. 高可靠：多地域多副本，数据可靠性远高于普通企业机房

自建机房优势：

  1. 数据可控：数据完全存储在自有服务器，满足金融、政府等强合规要求
  2. 定制化：可根据业务需求定制硬件架构、网络拓扑，适配特殊需求
  3. 低时延：本地网络访问，时延远低于公有云，适合实时性要求极高的业务
  4. 长期成本：业务稳定运行3-5年以上，长期总成本低于云计算
第3题（思考讨论题）：如果你要设计一个个人网站,需不需要云计算技术?

结论：需要，云计算是个人网站的最佳选择。

核心理由：

  1. 成本优势：个人网站访问量低，云服务器包月仅几十元，远低于购买物理服务器的成本
  2. 弹性适配：个人网站流量波动大（如文章爆火时流量暴增），云计算可弹性扩容，避免服务器崩溃
  3. 免运维：个人开发者通常没有专业运维能力，云厂商负责硬件维护、网络保障，用户只需关注网站本身
  4. 快速上线：云服务器分钟级开通，网站可快速部署上线，无需复杂的硬件部署流程
  5. 高可用：云服务器提供快照、备份、自动恢复功能，避免个人服务器硬件故障导致网站下线

五、发展历程 思考讨论题（第12-14题）
第12题（思考讨论题）：网络带宽、存储成本、CPU性能这三者,哪个对云计算普及影响最大?为什么?

结论：网络带宽对云计算普及影响最大。

核心原因：

  1. 云计算的本质是"网络即计算机"：所有计算与存储都在云端，用户通过网络访问，带宽是连接用户与云的桥梁，没有足够的带宽，云服务的体验会极差
  2. 带宽是用户体验的瓶颈：存储成本下降、CPU性能提升都是云端的优化，但用户实际体验直接受限于本地到云端的网络带宽，带宽不足会导致卡顿、延迟、加载慢
  3. 历史验证：云计算真正普及是在2010年后光纤宽带与4G网络大规模部署之后，在此之前带宽不足，云服务无法大规模应用
  4. 存储与CPU是内部优化：存储成本下降、CPU性能提升只是降低云厂商的成本，而带宽提升直接改善用户体验，是云计算从"可用"到"好用"的关键
第13题（思考讨论题）：未来算力会不会再次回到"本地为主、云为辅"?比如端侧AI变强、本地算力足够?

结论：不会，云边协同是长期趋势，云端仍将占据主导地位。

核心论述：

  1. 训练永远依赖云端：AI大模型训练需要数千甚至数万张GPU并行，需要PB级训练数据，这是端侧算力永远无法实现的，训练必须在云端完成
  2. 推理实现云边分工：简单、低延迟的推理任务可以放在端侧执行，但复杂、高精度的推理任务仍需云端大模型支持
  3. 数据协同需要云端：多设备数据同步、多用户协同、数据长期存储必须依赖云端，端侧无法实现跨设备、跨用户的数据协同
  4. 成本与效率优势：云端规模效应带来的成本优势是端侧无法比拟的，云端算力的单位成本远低于端侧
  5. 总结：未来是"云端训练+端侧推理"的云边协同模式，云端负责大规模计算与存储，端侧负责低延迟推理，不会回到本地为主的模式。
第14题（思考讨论题）：回顾云计算发展历史,你认为下一个十年,云计算最大的变革会是什么?

下一个十年云计算的核心变革方向：

第68题（计算题）：四个主频3.0GHz的CPU，每个四核，超线程。可虚拟多少VCPU和多少GHz总资源?

【完整计算过程】

第一步：计算物理核心总数

物理核心数 = CPU个数 × 每CPU核心数 = 4 × 4 = 16个物理核心

第二步：计算VCPU数量（超线程技术1核=2线程=2个VCPU）

VCPU总数 = 物理核心数 × 2 = 16 × 2 = 32个VCPU

第三步：计算总算力资源

总算力 = VCPU总数 × 单核心主频 = 32 × 3.0GHz = 96GHz

考点说明：超线程技术是考试常考点，记住公式：超线程下 1物理核心 = 2个逻辑核心（VCPU）

第69题（计算题）：2.8GHz单物理机，VM A/B/C份额1000/2000/4000，预留700/0/0。满负载每台分配多少?

第一步：先分配VM A的预留资源

剩余可分配资源 = 2800MHz - 700MHz = 2100MHz

第二步：计算份额总和

总份额 = 1000 + 2000 + 4000 = 7000份

  • VM B分配 = 2100 × (2000/7000) = 600MHz
  • VM C分配 = 2100 × (4000/7000) = 1200MHz

第2题：什么是大数据的4V，这些特征对大数据计算过程带来什么样的挑战？

  1. Volume（数据量大）：从TB级到PB、EB级，数据规模远超传统数据

    挑战：需要强大的数据挖掘和机器学习算法，从海量数据中提取价值

大数据技术的两大核心是：分布式存储和分布式计算

1. 分布式存储技术

核心思想：将海量数据分散存储在多台普通服务器上，而不是一台大型服务器上
典型技术：HDFS（Hadoop分布式文件系统）
举例：1PB的用户行为数据，无法存储在一台服务器上，HDFS将数据分成128MB的块，分散存储在100台服务器上，同时每个块保存3个副本，保证数据可靠性。

2. 分布式计算技术

核心思想：将一个大计算任务拆分成多个小任务，分发到多台服务器并行计算，最后汇总结果
典型技术：MapReduce、Spark
举例：统计10亿条日志的PV/UV，MapReduce将任务拆分成Map阶段（每台服务器统计自己本地数据）和Reduce阶段（汇总所有服务器的结果），100台服务器并行计算，几小时就能完成，而单机需要几个月。

总结：

分布式存储解决了大数据"存不下"的问题，分布式计算解决了大数据"算不动"的问题，二者共同构成了大数据技术的基石。

P(Hot|No) = 2/5 = 0.4 （No的5个样本中，Hot出现2次）
